Welcome — this is your team's project repo

Push your work to main and HackOnVibe builds & hosts it automatically at your team subdomain. Here's exactly how we pick your build.

How we detect & deploy your project

The pipeline reads your repo and chooses one of four builds — you don't configure anything, just push what you built:

  • 🟢Plain website

    Trigger: no build step (no package.json, or no build script). We serve your files as-is — your index.html shows up on Cloudflare Pages.

  • 🟡Built (static) app

    Trigger: a static framework — React/Vite/Astro, or Next.js with output: 'export'. We run npm run build and publish the output (dist/build/out/…) to Cloudflare Pages.

  • 🔵Server app (SSR)

    Trigger: Next.js in SSR mode (has next, no output: 'export') — it needs a live Node runtime. We build and run it in an isolated Docker container behind a tunnel.

  • 🟠Node backend

    Trigger: a non-Next Node server — a server dependency (express/fastify/…) or a server/start script that runs node. We host it in a Docker container same-origin: your frontend + your API on one domain (/api is proxied to your server). Need secrets? Add them in your cabinet at hackonvibe.com/me.

🧩 Using a different stack?

If your build isn't one of the four above (a different language, runtime, or setup), we'll still host you — but two steps first: (1) push the exact build you plan to use to GitHub (so we can see it), then (2) tell the organizers on Discord. We'll try to put together a suitable Docker container for you.

⚠️ Don't delete these — the build needs them

Keep .github/workflows/deploy.yml (our deploy pipeline — it's what puts your site live) and, if your app has one, package.json. Removing them breaks auto-deploy, and your subdomain won't update.

⏱️ After you push to main, your site goes live in about 2–5 minutes. Still not live after 5? Something went wrong — tell the organizers on Discord so we can help. · Automating your Git pushes with AI: gh CLI quickstart.
